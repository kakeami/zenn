## はじめに

私には二人の子どもがいます。どちらも男の子です。

男の子特有の性質[^text]なのか、私の気質を受け継いだのかわかりませんが、二人ともとにかく迷路が大好きです。
書店で買ったものはすぐに解いてしまうので、コピー用紙に手描きで作問することもしばしば。
数えていませんが、確実に100問は作成したと思います。

普通の迷路に飽きてしまった二人のために、ある時から **多面体迷路** を作成し始めました。
「多面体迷路」は私の造語であり、文字通り多面体の表面に展開する迷路を指します。
1面で完結する迷路と異なり、常に全体を見渡せないため、難易度が跳ね上がります。

技術スタックは Vanilla TypeScript + Vite + [Three.js](https://threejs.org/) で、UIフレームワークは使っていません。コードは役割ごとに3つの層に分かれています。

- **`core/`（数理コア）**: 多面体の定義・面グリッド・グラフ・迷路アルゴリズムなど、純粋な計算部分です。設計上の肝は、ここが **DOM・ブラウザAPIに一切依存しない** こと。おかげで数理ロジックを [Vitest](https://vitest.dev/) で隅々までユニットテストできます。乱数はシード付きの mulberry32 を使っているので、同じ `seed` なら必ず同じ迷路が再現されます。出力はゴールデンフィクスチャで固定し、リファクタしても結果が変わらないことを担保しています。
- **`render/`（描画）**: Three.js で3Dビューア（シーン・カメラ・OrbitControls）を、別経路でSVGの展開図を描き、jsPDF + svg2pdf.js でPDFに書き出します。3D表示と展開図PDFは同じ迷路データを使い回しています。
- **`ui/`（UI・状態）**: パラメータパネルと、設定をURLクエリ文字列に相互変換する param-codec、それらを配線する app です。設定がすべてURLに載るので、URLを共有すればまったく同じ迷路が開きます。

とくに「`core/` がDOMに依存しない」おかげで、多面体を141種類まで増やす作業を、ブラウザを立ち上げずテストだけで検証しながら進められたのが効きました。

### 使い方

[ツール本体](https://kakeami.github.io/polyhedral-maze/)をブラウザで開くと、3Dビューアと操作パネルが現れます。ラベルは英語ですが、画面に並ぶ項目は次のとおりです（カッコ内は初期値）。

| 画面の項目 | 役割 | 値 |
|---|---|---|
| **Category** | 多面体の一覧を分類で絞り込む（`All categories` で全表示） | 7分類 + All categories |
| **Shape** | 多面体そのもの。直下に「分類 · 面構成 · 面数」を表示 | 全141種（Icosahedron） |
| **Algorithm** | 迷路生成アルゴリズム | Kruskal / DFS / Wilson（DFS） |
| **n (resolution)** | 面の1辺あたりのグリッド解像度（スライダー） | 2〜12（9） |
| **k (passages)** | 隣接する面の1辺あたりの通路数（スライダー） | 1〜4（3） |
| **Seed** | 乱数シード | 0〜999999（42） |
| **Warp** | ワープトンネルの有無（チェックボックス） | off |
| **Show solution** | 解答パスを重ねて表示（チェックボックス） | off |
| **Auto-rotate** | 3Dビューアの自動回転（チェックボックス） | on |

パネル下部には3つのボタンがあります。

- **Random**: `Seed` を乱数で振り直し、別パターンを引く
- **Copy URL**: いまの設定を反映した共有URLをクリップボードにコピー
- **Export PDF**: 展開図PDF（問題＋解答）を書き出す（処理中は `Exporting...` 表示）

迷路を生成するたびに、その下に**メトリクス**が出ます — `Cells`（総セル数）/ `Solution`（解の長さと全体に占める割合）/ `Dead ends`（行き止まりの数と密度）/ `Face crossings`（面をまたぐ回数）/ `Faces visited`（訪れた面数 / 全面数）、ワープを使った場合は `Warp: used`。3D表示の色は凡例のとおりで、🟢 **Start**・🔴 **Goal**・🟡 **Warp** です。

`Category` と `Auto-rotate` を除く設定は、URLのクエリ文字列に保存されます（`shape` / `n` / `k` / `algo` / `seed` / `warp` / `solution`。初期値と同じ項目は省略され、`algo` は小文字になります）。なので **Copy URL** で得たURLを共有すれば、相手の画面にもまったく同じ迷路が開きます（例: `?shape=snub-dodecahedron&n=3&algo=wilson&seed=7&warp=1`）。

3Dビューアはドラッグで回転、ホイールでズームできます。構成が決まったら **Export PDF** で展開図（問題＋解答）が手に入ります。あとはこれを印刷して、切って、組み立てれば——という話を次章でやります。

「多面体迷路」は私の造語ですが、立体の表面に迷路を張る試みそのものには先行例があります。調べた範囲で関連するものを、ジャンルごとに挙げておきます。

**平面グリッドの迷路生成**
完全迷路（＝グリッドグラフの全域木）の生成は古典的なテーマで、DFSバックトラッカ・Kruskal・Prim・Aldous–Broder・Wilsonなどが体系的に整理されています（Jamis Buck『Mazes for Programmers』）。[Berthier (2021)](https://uca.hal.science/hal-03174952/document) はトーラス・円柱・メビウスの帯といった位相的バリエーションまで扱うサーベイですが、多面体表面そのもののモデルは提示していません。

**立方体表面の迷路**
立方体に限れば実装例が複数あります。[Vijayaraghavan & Krishnamurthy (2006, UC Berkeley CS285)](https://people.eecs.berkeley.edu/~sequin/CS285/PROJ_06/Adarsh--Athulan/AA_report/report.html) は転がるボールの迷路をMATLABで生成し3Dプリントまで、[carvey_cube_maze (2019)](https://github.com/mit-mit-randomprojectlab/carvey_cube_maze) はPythonでSVGの展開図を出力します。[Walter Pullen の Daedalus](https://www.astrolog.org/labyrnth/algrithm.htm) には立方体に矩形グリッドを巻く "planair" モードがあります。いずれも立方体専用で、面間の通路数制御やグラフ理論的な一般化はありません。

**一般の曲面・多面体の迷路**
[Li & Mount (2017)](https://www.cs.umd.edu/~mount/Indep/Xue-Li/xue-scholarly-paper.pdf) は球面上にVoronoi分割で（不規則な）セルを敷いて迷路を生成します。[Pedersen & Singh (2006) "Organic Labyrinths and Mazes"](https://doi.org/10.1145/1124728.1124742) は曲線の成長シミュレーションで有機的な迷宮を描きます。どちらもグリッドベースの多面体面迷路ではありません。[Matthew Arcus の PolyJS](https://github.com/matthewarcus/polyjs) は任意の多面体（OFFファイル）に迷路を作れますが、**1面＝1セル**として扱うため、たとえば二十面体でも20セルしかなく、パズルとしては粗すぎます。

**ホロノミー迷路**
[Henry Segerman のホロノミー迷路](https://mathstodon.xyz/@henryseg/106281620265498101) は、辺をまたぐとトークンの向きが変わる、多面体上の向き依存パズルです。数学的には深い対象ですが、全域木でも唯一解でもグリッド分割でもなく、本稿の完全迷路とは別物です。

**難易度の指標**
迷路の難しさを定量化する研究もあります（McClendon (2001, Bridges) の通路のうねり、Buck の行き止まり「数」、[Kim & Crawfis (2018)](https://doi.org/10.1109/ICCBD.2018.8693301) の解経路評価など）。ただし、面をまたぐ回数のような多面体表面に固有の難しさを捉えた指標は見当たりませんでした。

**特許**
[US Patent 4,008,895 (1977)](https://patents.google.com/patent/US4008895A/en) は、多面体の各面に通路を設け、共有辺でつないだ転がるボールの物理パズルです。物理的な物品の特許で、自動生成やアルゴリズムは対象外です。

まとめると、**立方体止まり／面＝1セル／グリッドなし**のいずれかというのが既存の状況で、任意の凸多面体に対して面ごとのグリッドを張り、面間の通路数を制御しながらグリッドベースの完全迷路を生成する、という枠組みは、調べた範囲では見当たりませんでした。
